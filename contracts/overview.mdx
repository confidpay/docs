# Smart Contracts Overview

ConfidPay's smart contract layer handles encrypted payroll storage, computation triggers, and payment release logic.

## Contract Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                    ConfidPay.sol                            │
│  (Main payroll contract with FHE encrypted storage)        │
│                                                             │
│  ├── Payroll storage (encrypted mappings)                  │
│  ├── ACL management (allowThis, allowSender, allow)        │
│  ├── Release conditions (time, milestones, vesting)        │
│  └── Integration hooks (Privara, external protocols)       │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                  External Dependencies                       │
│                                                             │
│  ├── @fhenixprotocol/cofhe-contracts/FHE.sol               │
│  │   └── Encrypted types & operations                      │
│  ├── @openzeppelin/contracts/access/Ownable.sol           │
│  │   └── Owner-based access control                        │
│  └── Privara SDK integration                               │
│       └── Confidential transfer patterns                    │
└─────────────────────────────────────────────────────────────┘
```

## Core Components

### Payroll Structure

```solidity
struct Payroll {
    euint256 encryptedAmount;        // Monthly/yearly salary
    euint256 encryptedNextRelease;    // Next unlock timestamp
    euint256 encryptedVestingRate;    // Vesting percentage
    euint256 encryptedMilestoneHash;  // Milestone conditions
    bool isActive;                     // Payroll status
}
```

### Key Functions

| Function | Description | Access |
|----------|-------------|--------|
| `createPayroll()` | Initialize encrypted payroll for recipient | Owner only |
| `releasePayment()` | Trigger payment release if conditions met | Anyone |
| `updatePayroll()` | Modify encrypted payroll parameters | Owner only |
| `cancelPayroll()` | Deactivate payroll and escrow remaining | Owner only |
| `getPayrollInfo()` | View encrypted payroll metadata | Recipient/Owner |

### Encrypted Storage

All sensitive data is stored as `euint256`:

```solidity
mapping(address => Payroll) public payrolls;
mapping(address => euint256) public encryptedBalances;
```
