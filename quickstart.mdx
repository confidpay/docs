# Getting Started

**This guide walks you through setting up your development environment and deploying the ConfidPay skeleton contract.**

## Prerequisites

- Node.js v19+
- Git
- Basic Solidity knowledge
- Testnet ETH from faucet (Base Sepolia or Arbitrum Sepolia)

## Step 1: Clone the Starter Project

```bash
git clone https://github.com/confidpay/cofhe-hardhat-starter
cd cofhe-hardhat-starter
npm install
```

The starter includes:
- Hardhat configuration with Fhenix plugin
- CoFHE mock environment for local testing
- Example contracts demonstrating FHE types

## Step 2: Configure Environment

Create `.env` from the example:

```bash
cp .env.example .env
```

Add your configuration:

```bash
# Required
PRIVATE_KEY=0x...           # Deployer wallet private key

# Testnet RPCs (choose one or both)
BASE_SEPOLIA_RPC=https://sepolia.base.org
ARBITRUM_SEPOLIA_RPC=https://sepolia.arbitrum.io/rpc
```

**Get testnet ETH:**
- [Base Sepolia Faucet](https://www.coinbase.com/faucets/base-ethereum-sepolia-faucet)
- [Arbitrum Sepolia Faucet](https://www.alchemy.com/faucets/arbitrum-sepolia)

## Step 3: Explore the Contract Structure

```
contracts/
├── ConfidPay.sol           # Main payroll contract (your implementation)
├── FHEHelper.sol           # ACL utilities
└── mocks/
    └── MockUSDC.sol        # Test token
```

Key FHE types you'll work with:

```solidity
// contracts/ConfidPay.sol
import "@fhenixprotocol/cofhe-contracts/FHE.sol";

contract ConfidPay {
    // Encrypted 256-bit integer - stores salary, amounts, timestamps
    euint256 public encryptedAmount;
    
    // Encrypted boolean - stores release conditions
    ebool public encryptedCanRelease;
    
    // Encrypted address - stores recipient
    eaddress public encryptedRecipient;
}
```

## Step 4: Deploy the Skeleton Contract

Deploy to Base Sepolia:

```bash
npx hardhat run scripts/deploy.ts --network baseSepolia
```

Deploy to Arbitrum Sepolia:

```bash
npx hardhat run scripts/deploy.ts --network arbitrumSepolia
```

Expected output:
```
ConfidPay deployed to: 0x...
```

Save this address — you'll need it for the frontend integration.

## Step 5: Test with Local Mock

Before testnet deployment, test locally with the CoFHE mock:

```typescript
// test/ConfidPay.test.ts
import { expect } from 'chai';
import { FhenixHelpers } from '@fhenixprotocol/contracts';

describe('ConfidPay ACL Setup', function () {
  it('should set correct ACL permissions', async function () {
    const { confipay, owner, employee } = await loadFixture(deployFixture);
    
    // Encrypt values locally for testing
    const encryptedAmount = FhenixHelpers.encrypt(5000000);
    
    await confipay.createPayroll(
      employee.address,
      encryptedAmount,
      // ... other encrypted params
    );
    
    // Verify ACL was set correctly
    const hasACL = await confipay.verifyACL(employee.address);
    expect(hasACL).to.be.true;
  });
});
```

Run tests:

```bash
npx hardhat test
```

## Step 6: Frontend Integration

Set up the Next.js frontend:

```bash
npx create-next-app@latest my-confidpay-frontend
cd my-confidpay-frontend
npm install @cofhe/react viem
```

Configure the provider:

```tsx
// app/providers.tsx
import { CofheProvider } from '@cofhe/react';
import { baseSepolia } from 'viem/chains';
import { http } from 'viem';

export function Providers({ children }: { children: React.ReactNode }) {
  return (
    <CofheProvider
      config={{
        chains: [baseSepolia],
        transports: {
          [baseSepolia.id]: http(),
        },
        contractAddress: '0x...', // Your deployed address
      }}
    >
      {children}
    </CofheProvider>
  );
}
```

## Step 7: Encrypt and Submit

The core workflow — all encryption happens client-side:

```tsx
// app/payroll/create.tsx
import { useEncrypt, useWrite } from '@cofhe/react';

function CreatePayroll() {
  const { encrypt } = useEncrypt();
  const { write } = useWrite();

  const createEncryptedPayroll = async () => {
    // 1. Encrypt all sensitive data BEFORE sending to contract
    const encryptedAmount = await encrypt(5000000); // 5 USDC
    const encryptedReleaseTime = await encrypt(
      Math.floor(Date.now() / 1000) + 30 * 24 * 60 * 60 // 30 days
    );
    const encryptedVestingRate = await encrypt(100); // 100%

    // 2. Submit transaction with encrypted values
    // On-chain: only euint256 is stored, plaintext is never visible
    await write('createPayroll', [
      employeeAddress,
      encryptedAmount,
      encryptedReleaseTime,
      encryptedVestingRate,
      0, // milestone hash
    ]);
  };

  return <button onClick={createEncryptedPayroll}>Create Payroll</button>;
}
```

## Understanding the Data Flow

```
1. ENCRYPT (client-side)
   salary = 5000000
   ↓ useEncrypt()
   encryptedSalary = "0xencrypted..."

2. SUBMIT (on-chain)
   tx = createPayroll(recipient, encryptedSalary, ...)
   → Contract stores euint256, not plaintext
   → Block explorer shows only: "0xencrypted..."

3. COMPUTE (CoFHE coprocessor)
   Contract requests: Is block.timestamp >= releaseTime?
   → Comparison happens on encrypted values
   → Result is also encrypted

4. RELEASE (on-chain)
   Payment transferred (still encrypted to outside observers)

5. DECRYPT (client-side only)
   recipient.decrypt(payment)
   → salary = 5000000 (plaintext revealed only to recipient)
```

## Common Pitfalls

<AccordionGroup>
  <Accordion title="Missing ACL permissions">
    **Never skip ACL setup.** For every encrypted field:

    ```solidity
    // ❌ WRONG - Funds locked forever
    p.encryptedAmount = FHE.asEuint256(encryptedAmount);

    // ✅ CORRECT
    p.encryptedAmount = FHE.asEuint256(encryptedAmount);
    FHE.allowThis(p.encryptedAmount);        // Contract can access
    FHE.allow(p.encryptedAmount, recipient);  // Recipient can decrypt
    ```
  </Accordion>
  <Accordion title="Decimal handling">
    USDC uses 6 decimals. 5 USDC = 5000000, not 5.
  </Accordion>
  <Accordion title="Timestamps">
    Use Unix timestamps (seconds), not milliseconds. `Date.now() / 1000`, not `Date.now()`.
  </Accordion>
</AccordionGroup>
