# Quickstart

**This guide walks you through setting up your development environment and deploying ConfidPay.**

## Prerequisites

- Node.js v18+
- Git
- Testnet ETH from faucet (Arbitrum Sepolia recommended)

## Step 1: Clone the Starter Project

```bash
git clone https://github.com/confidpay/ConfidPay-Starter
cd ConfidPay-Starter/cofhe-hardhat-starter
npm install
```

The starter includes:
- Hardhat configuration with Fhenix plugin
- CoFHE mock environment for local testing
- ConfidPay contract with vesting and milestones
- 26 passing tests

## Step 2: Configure Environment

Create `.env` from the example:

```bash
cp .env.example .env
```

Add your configuration:

```bash
# Required
PRIVATE_KEY=0x...                       # Deployer wallet private key

# Arbitrum Sepolia (recommended)
ARBITRUM_SEPOLIA_RPC=https://sepolia.arbitrum.io/rpc
```

**Get testnet ETH:**
- [Arbitrum Sepolia Faucet](https://www.alchemy.com/faucets/arbitrum-sepolia)

Key FHE patterns:

```solidity
// contracts/ConfidPay.sol
import "@fhenixprotocol/cofhe-contracts/FHE.sol";
import "@fhenixprotocol/cofhe-contracts/operations/operations.sol";

contract ConfidPay {
    // Encrypted salary - stored as euint128
    euint128 public encryptedAmount;
    
    // ACL setup - grant access to employee
    FHE.allowThis(encryptedAmount);
    FHE.allow(encryptedAmount, employee);  // NOT allowSender!
}
```

## Step 6: Frontend Integration

Install dependencies:

```bash
npm install @cofhe/sdk viem wagmi
```

Create the Fhenix client:

```typescript
// lib/fhenix.ts
import { FhenixClient } from '@cofhe/sdk/web';

let client: FhenixClient | null = null;

export function getFhenixClient(): FhenixClient {
  if (!client) {
    client = new FhenixClient();
  }
  return client;
}
```

Create payroll with encryption:

```typescript
// components/CreatePayroll.tsx
import { getFhenixClient } from '@/lib/fhenix';

async function createEncryptedPayroll() {
  const client = getFhenixClient();
  
  // 1. Encrypt all sensitive data BEFORE sending to contract
  const encryptedAmount = await client.encrypt(5000000); // 5 USDC
  const encryptedReleaseTime = await client.encrypt(
    Math.floor(Date.now() / 1000) + 30 * 24 * 60 * 60 // 30 days
  );
  const encryptedVestingRate = await client.encrypt(100); // 100%

  // 2. Submit transaction with encrypted values
  await write('createPayroll', [
    employeeAddress,
    { publicAmount: encryptedAmount },      // InEuint128 format
    { publicAmount: encryptedReleaseTime },
    { publicAmount: encryptedVestingRate },
  ]);
}
```

## Understanding the Data Flow

```
1. ENCRYPT (client-side)
   salary = 5000000
   ↓ client.encrypt()
   encryptedSalary = "0xencrypted..."

2. SUBMIT (on-chain)
   tx = createPayroll(recipient, {publicAmount: encryptedSalary}, ...)
   → Contract stores euint128, not plaintext
   → Block explorer shows only: "0xencrypted..."

3. COMPUTE (CoFHE coprocessor)
   Contract requests: Is block.timestamp >= releaseTime?
   → Comparison happens on encrypted values
   → Result is also encrypted

4. ESCROW (ReineiraOS)
   Payment held in escrow until conditions met

5. DECRYPT (client-side only)
   client.decrypt(payment)
   → salary = 5000000 (plaintext revealed only to recipient)
```

## Common Pitfalls

<AccordionGroup>
  <Accordion title="Using allowSender instead of allow">
    **Never use `FHE.allowSender()` for employee access.** It grants admin access, not employee access.

    ```solidity
    // ❌ WRONG - Employee cannot decrypt
    FHE.allowSender(p.encryptedAmount);

    // ✅ CORRECT - Employee can decrypt their own data
    FHE.allow(p.encryptedAmount, employee);
    ```
  </Accordion>
  <Accordion title="Using if() with FHE comparisons">
    **Never use `if (FHE.eq(...))` or `if (FHE.lt(...))`.** Use `FHE.select()` instead.

    ```solidity
    // ❌ WRONG - Compilation will fail
    if (FHE.eq(value, FHE.asEuint128(0))) {
        revert("Zero");
    }

    // ✅ CORRECT - Use FHE.select()
    euint128 result = FHE.select(
        FHE.eq(value, FHE.asEuint128(0)),
        FHE.asEuint128(0),
        value
    );
    ```
  </Accordion>
  <Accordion title="Decimal handling">
    USDC uses 6 decimals. 5 USDC = 5000000, not 5.
  </Accordion>
  <Accordion title="Timestamps">
    Use Unix timestamps (seconds), not milliseconds. `Date.now() / 1000`, not `Date.now()`.
  </Accordion>
  <Accordion title="Type conversion">
    Use `InEuint128.Constructor` for function parameters and `FHE.asEuint128()` to convert.
  </Accordion>
</AccordionGroup>
